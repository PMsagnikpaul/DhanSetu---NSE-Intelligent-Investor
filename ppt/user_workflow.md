# User Workflow: NSE Intelligent Investor

## Overview
The NSE Intelligent Investor platform is designed to provide retail investors with institutional-grade quantitative analysis. The workflow is entirely unified within a single, interactive frontend dashboard (`nse_intelligent_investor.html`), fed by a FastAPI Python backend.

## Step-by-Step Workflow

### 1. Initialization and Data Integrity Verification
**Action:** The user launches the application (opens the HTML file in a browser or visits the deployed URL).
**Background Process:**
- The system connects to the backend and triggers the `api.py` `lifespan` event.
- **QuantVault Regime Engine (Module 4)** loads aligned data (prices, yields, sentiment) and generates a SHA-256 data integrity hash to simulate a blockchain ledger, ensuring data provenance.
- The user is presented with a **Disclaimer Modal** explicitly stating the platform is algorithmic and not a registered financial advisor.

### 2. Regime Awareness and Market Context
**Action:** The user reviews the top header and dashboard summary.
**What happens:**
- The system displays the current market regime (e.g., "Bull", "Bear", "Sideways") based on the Hidden Markov Model (HMM) analysis.
- If market conditions are extreme, the **Circuit Breaker** intervenes, overriding the HMM to force a "Capital Preservation" mode and altering UI colors.

### 3. Scanning the Opportunity Radar (Module 1)
**Action:** The user navigates to the "Opportunity Radar" tab to discover new stock picks.
**What happens:**
- The system fetches daily signals extracted from Corporate Filings, Insider Trades, and Bulk Deals.
- These signals are passed through an **LSTM Anomaly Detector** to filter out noise, producing a ranked composite score.
- The user clicks on a high-scoring stock (e.g., RELIANCE) and reads a plain-English explanation generated by the LLM integration explaining *why* the stock is flagged.

### 4. Technical Validation via Chart Patterns (Module 3)
**Action:** Before deciding to buy, the user cross-references the Opportunity Radar pick with Technical Analysis.
**What happens:**
- The user searches for RELIANCE in the "Chart Patterns" module.
- The system algorithmically scans the stock's recent OHLCV data to detect 8 classical patterns (e.g., "Bullish Breakout", "Double Bottom").
- An **LSTM Continuation Scorer** calculates the probability (0-100%) that the detected pattern will succeed, while the historical Backtester shows the exact historical win-rate of that pattern for RELIANCE.

### 5. Portfolio Construction and Optimization (Module 2)
**Action:** Confident in the signals, the user adds the selected stocks to their intended portfolio universe in the "My Portfolio" settings and navigates to the "Portfolio Optimizer".
**What happens:**
- The user inputs their risk tolerance and investment horizon (e.g., 15 days).
- The system's **LSTM Predictor** forecasts the annualized returns for each stock.
- The **MPT Optimizer** computes the Efficient Frontier and outputs optimal capital allocation weights to maximize the Sharpe ratio.
- The system generates a "Rebalancing Proposal", telling the user exactly how many shares of each stock to BUY, SELL, or HOLD.

### 6. Managing Black Swan Events and Drawdowns
**Action:** A sudden macroeconomic shock occurs during the trading day.
**What happens:**
- The background **Circuit Breaker** detects a >3% daily drop and a VIX spike.
- A critical alert pops up on the user's screen. The Portfolio Damage Assessment estimates the user's current loss and checks if predefined stop-losses were breached.
- The user queries the **Loss Recovery Engine** to ask: "How long until my portfolio recovers?" The system scans historical instances of similar drops and returns a median recovery timeframe (e.g., "Historical median recovery for a -4% drop in TCS is 12 trading days").

## Example Scenario

**Meet Rahul**, an investor using the platform:
1. **Login:** Rahul opens the app. The Regime Header shows **"Bull Market - 85% Confidence"**.
2. **Discovery:** The Opportunity Radar flags **INFY** with a 92% score due to heavy insider buying and a positive corporate filing.
3. **Validation:** Rahul checks Chart Patterns. INFY shows an "Inverse Head & Shoulders" pattern. The LSTM Continuation Scorer gives it a 78% probability of success, and the backtested historical win rate is 65%.
4. **Optimization:** Rahul wants to allocate ₹1,00,000 between INFY, TCS, and RELIANCE. The Optimizer runs the LSTM-MPT pipeline and suggests an optimal weight of 45% INFY, 30% TCS, and 25% RELIANCE to maximize returns while maintaining a low CVaR (Conditional Value at Risk).
5. **Execution:** Rahul executes the trades on his broker based on the Rebalancing Proposal.
6. **Crisis Management:** Two weeks later, the market crashes by 4%. The platform's Circuit Breaker overrides the regime to **"Bear - Capital Preservation"**. Rahul's dashboard flashes red. The Recovery Engine tells him INFY typically takes 8 days to recover from such a drop, preventing him from panic selling.
